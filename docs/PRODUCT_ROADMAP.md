# Personal Trading Agent — 产品 Roadmap

> 创建于 2026-06-02。这是**产品/体验功能**的路线图，与策略迭代 backlog（见 memory `trading_next_week_todos.md`）是两条线：本文件管"产品能力"，那边管"选股/买卖逻辑参数"。

## 北极星

这套系统的独特定位是 **「AI 全自动执行 + 人在环监督/否决」**。
产品设计的唯一北极星：**让用户能在 ~10 秒内看懂 AI 在做什么、并决定要不要干预。**

每个功能的取舍都回答一句话：*它是否让"监督 AI"更快、更可信？*

## P0 — 核心能力（缺了就是盲操作）

### P0-1　K 线图 + 关键价位可视化　🥇「最大的 miss」　✅ 已上线（2026-06-02）
> 已实现：点持仓 symbol → 模态看日线 K 线（lightweight-charts）+ MA20/50 + 成交量 + 买入/止损/止盈/现价标注线。后端 `/api/ohlcv/{symbol}`（Alpaca）。
> 后续小优化：K 线偶尔加载 1-4s（Alpaca 延迟）→ 可加缓存提速。

**问题**：交易系统没有图 = 蒙眼信任 AI。看不到形态就无法判断"真突破 vs 诱多"，无法对 AI 信号做快速否决。
**范围**：
- 每只持仓 / 候选可点开日线 K 线（~90 个交易日）
- 叠加 MA20 / MA50、成交量
- 标注：买入点、止损线（−2×ATR 钳位）、止盈线（+10% 追踪激活）、当前价
- 复盘场景：平仓后能回看"买在哪、为什么破"
**数据**：Alpaca bars（扫描已在用，无需新源）。
**依赖**：无。**建议第一个做。**

### P0-2　决策可解释「决策卡」　🥈
**问题**：AI 给 BUY/SELL，但"为什么"散在一段 reason 文本里，用户无法信任或精准否决。
**范围**：一张可追溯的决策链卡片
`Maya regime → Scout Track/AI 评分 → Rex 入场门控（SPY>MA20 / 财报 / 漂移 / Gate A/B）→ 止损价计算`
每一步 ✓/✗ 可见，让用户看懂全过程、一键否决。
**价值**：这是 AI 交易产品**最核心的差异化** —— 不是黑箱给信号，而是可解释的副驾。
**数据**：各 agent 已产出（market_context / scan candidate fields / trade reason / gate 结果），主要是聚合 + 展示。

**阶段1 ✅ 已上线（2026-06-02）**：决策链 Maya 环境(regime/激进度) → Scout 选股(Track/AI分/RSI/量比/理由) → Rex 执行(信号/置信度/止损止盈)，嵌在持仓详情模态（K 线下方）。
**阶段2（待办）**：逐项门控 ✓/✗（SPY>MA20、财报、价格漂移、Gate A/B）。需给 Rex 执行路径（trade_agent run_agent / approve_trade）**结构化记录每个门控的通过/拒绝结果**到 trade 对象 —— 动交易路径，按 CLAUDE.md 先出计划。
**扩展（待办）**：详情模态目前仅持仓入口；**信号页（SignalsView）候选**也接入 K 线 + 决策卡（候选数据已含 track/ai_score/rsi/vs_ma20/reason/stop/target，可直接复用 CandleChart + DecisionChain）。

## P1 — 信任 + 风控

### P1-3　复盘进化时间线
**问题**：「亏损 → 根因 → 候选改进 → 版本验证」的学习闭环目前埋在 memory / 对话里，没产品化。用户看不到系统在进化。
**范围**：一条可见的策略进化轨迹 —— 每个版本（v1–v7…）改了什么、哪笔亏损触发的、上线后胜率/盈亏比变化。扩展现有「复盘」Tab（当前只有版本胜率对比）。
**数据**：versions.json + trade_history（by version/track）已有；"哪笔亏损触发"需补结构化关联。

### P1-4　风险体检视图
**问题**：风险信息散落（板块集中度、单票超 8%、现金、距熔断）。盘前难一眼判断健康度。
**范围**：一屏风险仪表 —— 板块集中度（半导体等）、单票超限预警、现金/储备、回撤、距熔断距离。盘前/盘中一眼看要不要干预。
**数据**：positions / budget / circuit_breaker / sector 已有。
